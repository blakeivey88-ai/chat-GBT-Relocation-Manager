# Viral Score Workflow — Relocation Manager

## Goal

Before posting social content, run it through a viral-score tool and revise until it reaches **80+**.

Primary tool:

- Will It Go Viral / X Viral Score Analyzer: `willitgoviral.xyz`

## Rule

No planned social post should be published until it hits:

- **80+ score** for X/Twitter-style posts
- or Ralph/Felix documents why we are posting anyway, for example: important announcement, launch update, or non-viral operational post

## Workflow

1. Draft the post.
2. Run the post through the viral-score tool.
3. If score is below 80:
   - identify weak hook
   - strengthen emotion / clarity / curiosity
   - remove fluff
   - improve CTA
   - make the first line sharper
4. Re-score.
5. Save the final score next to the post in the weekly content pack.
6. Only then publish or ask Blake to publish.

## Suggested score fields for weekly packs

Add this under each post:

```md
Viral score: TBD / 100
Status: Needs scoring | Approved 80+ | Posted anyway because ____
Revision notes:
```

## Ralph/Felix division

- Ralph: strategy, brand voice, captions, launch announcements, conversion focus.
- Felix, if available later: viral scoring/rewrite pass, hook testing, alternate variants.

## Important note

Will It Go Viral appears strongest for X/Twitter-style text posts. For Instagram/Reels, use it as a hook/caption filter, not as a perfect prediction of actual reach.
